Semantic elements vs CSS implementations

It's often not very difficult to re-implement the styling of an HTML element with CSS; <blockquote> for instance

With CSS...

But semantic (and most other) elements carry additional ARIA role information. The role attribute is available, but is often an incomplete or poor substitute for a semantic element's built in accessibility features.

tl;dr: Prefer actual semantic elements to re-implementations. This is doubly true for form elements.
